Django Collada Client Examples

Django Collada is an open source project. It provides a simple way of working with COLLADA-models in a Django context. It provides a way to upload zipped collada archives (ie. .dae-file and related assets) to a Django web application that can serve the uploaded files in an accessible manner for visualisation on the client side.

In addition, the scenes can easily be set up (grids, axes, background colors etc.) using a provided configuration object for each supported visualisation technique.

How it works

Django Collada can import a COLLADA file to a Django project and then serve the dae and related assets statically. The COLLADA model is not converted to a full Django persistent object by any means. We instead emply the excellent pycollada library to access the mesh in detail.

Installation

  1. pip install django_collada
  2. Optional: If you want to have this page accessible in your project, ie. for playing with the examples provided you need to add the django_collada application to your project, add the following line to your urls.py:

    urlpatterns += (url(r'^django_collada', include('django_collada.urls')),)

    3. Also add django_collada to your djangos settings.py, INSATLLED_APPS

Beyond the examples

You should extend the django_collada.models.ColladaScene model class in your own applications and go from there to create your views and logic around the collada model. Any of the visualisation examples above can be used as a starting point or may serve well for simpler applications. If you need deeper interactions with scenes, you will of course need to dive into the WebGL framework (ie. three.js) of choice.

Suggested improvements

Django collada was developed as a component to a publicly funded project with a few specific requirements in mind so there are loads of possibilities of improvements. The ideas could be incorporated into the django_collada project, or in your own implementation.

Full CRUD
Would be cool to provide the full work flow of loading a model into the browser, change some properties (like positions, color, lighting etc.) and save it to the backend.
Data driven properties
Let some property of a 3D-model change depending on some underlying data structure, ie. by periodic updates through a JSON service.
Multi model scenes
Should not be too far away. Import a few models and then load them all into the same scene.
